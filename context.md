# Property Pulse ŌĆö Context

## What this is

Property Pulse ("PropPulse ŌĆö Tycoon Edition") is a single-user personal real
estate / rental property management app, built as an installable PWA. It is
in Thai (`lang="th"`) and targets a landlord tracking a small portfolio of
rental units (rooms/houses): rent collection, loan payments, expenses, and
basic tax reporting.

There is no backend, no build step, and no package manager. The entire
application is one static HTML file plus a service worker and manifest.

## Repo layout

```
index.html      Everything: markup, CSS (Tailwind utility classes + custom
                 styles in <style>), and all JS logic in a single <script>
                 block at the bottom of the file (~130KB).
manifest.json    PWA manifest (name, icons, theme colors, start_url).
sw.js            Service worker ŌĆö cache-first for the app shell, network-
                 first (cache fallback) for CDN assets.
icons/           App icons (icon.svg source, icon-192.png, icon-512.png,
                 icon-maskable.png) generated by make_icons.py.
make_icons.py    One-off Python/Pillow script that draws and exports the
                 PNG icon set from scratch. Not run automatically; rerun
                 manually if the icon design changes.
```

There are no `src/`, `tests/`, `dist/`, or config files (no `package.json`,
no bundler, no linter config). Everything ships as-is to a static host
(GitHub Pages ŌĆö see the share URL hardcoded in Settings:
`https://supasiao7896th.github.io/Property-Pulse/`).

## Runtime architecture (inside `index.html`)

- **Dependencies** are all loaded from CDNs, no `node_modules`:
  - Tailwind CSS (CDN build, used as a utility framework directly in markup)
  - Chart.js (`cdn.jsdelivr.net`) ŌĆö loaded but currently unused after the
    cash-flow chart and AI analyst card were removed (see git history).
  - Lucide icons (`unpkg.com`) ŌĆö re-run `lucide.createIcons()` after any DOM
    update that injects new `data-lucide` elements.
  - Google Fonts (Sarabun).
- **Content-Security-Policy** is declared via `<meta>` in `<head>`. It still
  whitelists `https://generativelanguage.googleapis.com` in `connect-src`
  even though the Gemini AI integration was deliberately removed (commit
  `6fb5b99` ŌĆö "remove: Gemini AI Engine"). Don't reintroduce AI calls without
  product direction; if you touch the CSP, keep it as tight as possible.
- **Persistence**: IndexedDB via a small `DBManager` module
  (`index.html` ~line 848), database name `PropertyPulseDB_V4`, version 2,
  with object stores: `rooms`, `transactions`, `settings`, `documents`
  (documents indexed by `roomId`). No `localStorage` is used for app data.
- **App module**: an IIFE called `App` (~line 889) owns all state and
  rendering. State shape: `{ rooms: [], transactions: [], viewMode: 'month' }`.
  Pattern is plain imperative DOM manipulation ŌĆö no virtual DOM, no
  framework. Views are toggled by adding/removing the `.active` class on
  `<section id="view-*">` elements (`switchView`, ~line 1369).
- **Views** (nav items, `data-view` attr): `dashboard`, `rooms`,
  `transactions`, `settings`.
  - **Dashboard**: month/year toggle, income/expense/net cards, occupancy,
    rent-collected progress, outstanding rent, per-unit performance list.

    target/balance (`calcLoanCurrentBalance`, `calcLoanSchedule` ŌĆö simple
    amortization estimate), move-out workflow, tenant history modal,

  - **Transactions** ("ÓĖ¬ÓĖĪÓĖĖÓĖöÓĖÜÓĖ▒ÓĖŹÓĖŖÓĖĄÓ╣üÓĖóÓĖüÓĖøÓĖŻÓĖ░Ó╣ĆÓĖĀÓĖŚ"): add income/expense entries

## Conventions to follow when editing

- This is intentionally a **zero-build, single-file** app. Don't introduce a
  bundler, framework, or `node_modules` unless explicitly asked ŌĆö that would
  be a major architecture change, not a small edit.
- All user-facing strings are in **Thai**. Match the existing tone/wording
  style when adding UI text.
- Money is always formatted via `formatMoney`/`formatMoneyShort`
  (`Intl.NumberFormat('th-TH', { style: 'currency', currency: 'THB' })`) ŌĆö
  don't hand-roll currency formatting.
- New DOM content built via `innerHTML`/template strings must escape
  user-provided values with `escHtml` to avoid XSS, since CSP allows
  `'unsafe-inline'` scripts but the app still must not trust stored data
  blindly when interpolating into markup.
- After injecting markup containing `data-lucide` icons, call
  `lucide.createIcons({root: ...})` (or globally) so icons actually render.
- IndexedDB is the single source of truth; in-memory `state` is a cache
  populated on `start()` and kept in sync by re-fetching/re-rendering after
  writes (`DBManager.put`/`remove` then `renderAll()` or targeted re-render).
- If you change the `documents`/`rooms`/`transactions`/`settings` schema,
  bump `CONFIG.DB_VERSION` and handle migration in `onupgradeneeded`.
- Service worker caches `./`, `./index.html`, `./manifest.json`,
  `./icons/icon.svg` as the app shell. If you rename/move any of those
  files, update `APP_SHELL` in `sw.js` and bump the `CACHE` name so old
  caches are purged on next activate.

## Testing

There is no automated test suite, linter, or CI config in this repo. The
only way to verify changes is manually: open `index.html` in a browser
(or serve the directory statically, since the service worker requires a
real origin to register) and exercise the affected view.
